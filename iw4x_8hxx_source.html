11 // MinGW and MSVC differ in how they treat exports for DLLs.
12 //
13 // MinGW (via GNU ld) allows producing a DLL with no explicitly exported
14 // symbols. In that model, DllMain is treated as a special entry point
15 // discovered by name and does not need to appear in the export table.
16 //
17 // MSVC's linker, on the other hand, requires at least one symbol to be
18 // placed in the DLL export table. If no symbol is explicitly marked for
19 // export (e.g. via __declspec(dllexport)), MSVC refuses to link the DLL
20 // and reports that no exported symbols exist.
21 //
22 // For this reason, we explicitly export DllMain when building with MSVC,
23 // even though this is unnecessary for MinGW and has no functional effect
24 // at runtime.
